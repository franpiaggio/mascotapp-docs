MascotApp · Quickstart

De cero a PR
en cuatro pasos.

La versión corta: clonás, levantás, creás tu feature y — si querés — dejás que el skill /ship-feature haga el resto.

01

Clonar el repo

Necesitás Node ≥ 20 (hay .nvmrc), npm y Docker. Después:

terminal
$ git clone https://github.com/franpiaggio/mascotapp.git
$ cd mascotapp
$ nvm use          # Node 20
$ npm install      # workspaces: instala api + web de una
02

Levantar el servidor

Un comando levanta base, API (con migraciones automáticas) y web:

terminal
$ cp .env.example .env    # editá JWT_SECRET: mínimo 32 caracteres
$ npm run docker:up

Solo Postgres en Docker; API y web con hot-reload:

terminal
$ npm run docker:db                # solo Postgres 16
$ cd apps/api && cp .env.example .env   # completá JWT_SECRET
$ npm run prisma:generate && npm run prisma:migrate && npm run prisma:seed
$ cd ../web && cp .env.example .env && cd ../..
$ npm run dev                      # API + Web en paralelo
  • Web: http://localhost:5173 · API: http://localhost:3001 · Swagger: /docs · Health: /health
👑 Admin
admin@mascotapp.local · Admin123!
🩺 Vet
vet@mascotapp.local · Vet12345!
⚠️

Si la API no arranca, casi siempre es el env: JWT_SECRET de menos de 32 caracteres o DATABASE_URL mal formado. La validación con Zod corta el boot y muestra el error en consola.

03

Crear una feature

1 · rama desde master
$ git checkout master && git pull
$ git checkout -b feat/mi-feature    # feat/ · fix/ · chore/

2 · Código. Copiá la estructura del módulo/feature más parecido:

  • API → módulo NestJS en apps/api/src/<dominio>/ (module + controller + service + dto). Si tocás schema.prisma: prisma:migrate + prisma:generate y commiteá la migración.
  • Web → carpeta en apps/web/src/features/ con página, hooks de React Query y primitivas de components/ui. Ruta en routes/router.tsx.
3 · checks (no hay CI: esto es el gate)
$ npm run lint && npm run typecheck && npm run test
4 · commit + push + PR
$ git commit -m "feat(api): add vaccines module"   # Conventional Commits
$ git push -u origin feat/mi-feature
$ gh pr create --base master \
    --title "feat(api): add vaccines module" \
    --body  "## Qué cambia ... ## Cómo probar ..."
💡

Nada de secretos ni .env en el diff. Si agregás variables de entorno, actualizá los .env.example y apps/api/src/config/env.ts.

04

Shipear con un skill

/ship-feature — el paso 3, automatizado

El repo incluye un skill de Claude Code en .claude/skills/ship-feature/ que hace todo el flujo de punta a punta: solo describís la feature.

rama desde master→ implementa con los patrones del repo→ lint + typecheck + tests→ commit convencional→ push + PR
dentro de claude (en la raíz del repo)
$ claude
> /ship-feature módulo de vacunas: CRUD en la API + página en el dashboard
  • Corre los checks antes de commitear — si fallan, los arregla antes de pushear.
  • Arma el PR con secciones Qué cambia y Cómo probar (con usuario seed incluido).
  • Al final te reporta rama, URL del PR y decisiones que tomó.
🔑

Requisitos: tener Claude Code instalado y gh autenticado (gh auth login). El skill se carga solo al abrir claude en el repo.
